Что выведет программа? Объяснить вывод программы. Объяснить внутреннее устройство интерфейсов и их отличие от пустых интерфейсов.

```go
package main

import (
	"fmt"
	"os"
)

func Foo() error { // error - тип интерфейса
	var err *os.PathError = nil
	return err
}

func main() {
	err := Foo()
	fmt.Println(err)
	// err - структура вида {type: *os.PathError, val: nil}
	// Динамическое значение err == nil, но его динамическим
	// типом является *os.PathError, а это значит, что err
	// представляет собой ненулевой интерфейс, содержащий
	// нулевое значение указателя
	fmt.Println(err == nil) // будет false
	// Сравнение даст true, если сделать следующее:
	var errA error // Объект интерфейса - никто не реализует
	// здесь errA - структура вида{type:nil, val:nil}
	// Нулевое значение интерфейса имеет и тип, и значение равные nil
	fmt.Println(errA == nil) // будет true
}
```

Ответ:
```
Значение интерфейсного типа (значение интерфейса) имеет 2 компонента: конкретный тип и значение этого типа.
Они называются динамическим типом и динамическим значением интерфейса.
```
